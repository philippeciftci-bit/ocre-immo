# Cause racine
Deux formulaires d'inscription parallèles non intégrés : (A) modale ocre.immo via `parts/signup-popup.php` (1 checkbox CGU+RGPD combinée), (B) wizard signup.ocre.immo via `/opt/ocre-app/inscription/` (CGU+RGPD séparés). Pas de SSO entre les deux. Après magic link, l'utilisateur retombait sur (B) au clic Oi Agent.

# Fix appliqué

## Form unifié auth.ocre.immo/signup
- `signup.html` existait déjà avec PhoneInput M86 (21 pays E.164) + email + prénom + nom + société + téléphone. Manquait juste la séparation CGU/RGPD.
- Split en 2 checkboxes obligatoires : `#cgu` + `#rgpd` (au lieu de 1 combinée). Validation front : submit disabled tant que les 2 ne sont pas cochées + autres champs valides.
- Label bouton : "Continuer" → "Recevoir mon lien d'accès" (spec).

## Backend double-lock
`/opt/ocre-auth/api/magic-link/request.php` :
- Lecture `rgpd_accepted` dans le body JSON.
- `$isFullSignup = ($first OR $last OR $phone)` → si full signup ET (!cgu OR !rgpd) → HTTP 400 `cgu_rgpd_required`.
- Email-only request (juste lien) reste sans check (cohérent : CGU déjà acceptés au signup initial).
- Persiste `rgpd_accepted_at` dans `auth_users` (colonne créée via ALTER TABLE IF NOT EXISTS).

## Nginx routes
- `auth.ocre.immo/signup` : nouvelle `location = /signup { try_files /signup.html =404 }` pour URL canonique sans .html.
- `signup.ocre.immo/*` : 3 `return 301 https://auth.ocre.immo/signup` (/, /inscription/, /signup). Sous-domaine conservé pour pas casser les liens email.

## Vitrine WP cleanup
`parts/signup-popup.php` : 412 lignes (modale + form + JS) → 30 lignes. Ne contient plus que `window.ocreSignupOpen = redirect_to_auth_signup`. Ancienne version archivée en `signup-popup.php.ARCHIVED-M_SIGNUP_UNIFIE-<TS>`. Tous les CTA `data-signup-trigger="agent"` redirigent vers `auth.ocre.immo/signup?app=agent&source=vitrine` (auto-bind DOMContentLoaded).
