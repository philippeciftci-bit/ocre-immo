# Cause racine (rappel diag M/41)
Safari iPad "Prevent Cross-Site Tracking" (ITP iOS 17+) bloque le cookie `ocre_jwt` `SameSite=Lax` sur fetch POST cross-subdomain depuis `ocre.immo` vers `auth.ocre.immo`, malgré le même eTLD+1. Cas A retombait systématiquement en cas B sur iPad Safari réel (curl + Playwright passaient car settings ITP par défaut moins stricts).

# Fix appliqué (3 endroits)

## 1. `auth-root/lib/auth_db.php` — `auth_set_cookies()` + `auth_clear_cookies()`
- `samesite => 'Lax'` → `samesite => 'None'` sur les 2 cookies `ocre_jwt` + `ocre_refresh` posés par `validate.php`.
- Cohérence sur `auth_clear_cookies` (logout) pour matcher les attributs au moment de l'expiration.
- `secure=true` + `httponly=true` + `domain=.ocre.immo` inchangés.

## 2. `auth-root/api/email-check.php` (ligne 70)
- Idem : `samesite => 'Lax'` → `'None'` sur le `setcookie('ocre_jwt'...)` inline déclenché par `email-check.php` (flow vitrine signup popup ancien).

**Smoke test post-fix** : `curl -I validate.php` confirme les en-têtes Set-Cookie :
```
ocre_jwt=...; expires=...; Max-Age=31536000; path=/; domain=.ocre.immo; secure; HttpOnly; SameSite=None
ocre_refresh=...; idem
```

# Mitigations CSRF (vérifiées toujours en place)
1. **JWT signature HMAC** : `jwt_decode` fait `hash_hmac` dans `/opt/ocre-auth/lib/jwt.php` (3 références au check signature).
2. **jti lookup `auth_sessions`** : `login.php` + `me.php` valident que le jti existe + `revoked_at IS NULL` + `expires_at > NOW()` (1 référence chacun).
3. **CORS Origin whitelist stricte** : `login.php` (2 références), allow-list `[ocre.immo, www.ocre.immo, auth.ocre.immo, agent.ocre.immo]` + regex tenant `^https://[a-z0-9-]+\.ocre\.immo$`. `Access-Control-Allow-Credentials: true` requis.
4. **Tenant redirect_url** : basé sur `users.slug` DB du user authentifié (3 références), pas d'input arbitraire user.

→ Sécurité OK : le passage à SameSite=None ne réduit pas la protection effective.
